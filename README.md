# Stefania — портфолио

Статический сайт: `index.html`, `styles.css`, `script.js`, папка **`assets/`** с изображениями.

## Где лежат фотографии

Все картинки для блоков портфолио подключены из папки:

**`assets/`** (в корне проекта, рядом с `index.html`)

Если на сайте после деплоя «битые» иконки вместо фото — почти всегда причина в том, что **папка `assets` не попала в Git / не загрузилась на Vercel**.

### Что сделать

1. Убедись, что в проекте есть папка `assets` с файлами `.png`.
2. При загрузке на GitHub выполни:
   ```bash
   git add assets/
   git status
   ```
   В списке должны быть все `.png` из `assets/`.
3. Закоммить и запушь:
   ```bash
   git commit -m "Add portfolio images"
   git push
   ```
4. В Vercel дождись нового деплоя (или нажми **Redeploy**).

### Проверка

Открой в браузере прямой URL к одному файлу (подставь свой домен Vercel и имя файла из `assets`):

`https://ТВОЙ-ПРОЕКТ.vercel.app/assets/ИМЯ-ФАЙЛА.png`

Если страница 404 — файл не задеплоен.
